  <p>The MedicationRequest and MedicationNotRequested profiles represent the positive and
    negative statements for a medication order.
  </p>

  <a name="examples"> </a>
  <h3>Examples</h3>
  <p>
  <table>
    <tbody>
      <tr>
          <td><a href="MedicationRequest-negation-example.html">Negation Example using Valueset</a></td><td>Medication not indicated; indicates that none of the medications in the given not done value set were requested.</td>
        </tr>
        <tr>
          <td><a href="MedicationRequest-negation-example-code.html">Negation Example using code</a></td><td>Medication not indicated; indicates that the medication specified by code was not requested.</td>
      </tr>
    </tbody>
  </table>
  </p>

  <p>Both of these example instances meet measure intent for exclusion as defined in the following illustrative CQL (informational; not part of conformance requirements):
  </p>

  <pre><code>define "Reason for Not Ordering Antithrombotic":
  ["MedicationNotRequested": "Antithrombotic Therapy"] NoAntithromboticDischarge
    where (NoAntithromboticDischarge.reasonCode in "Medical Reason"
      or NoAntithromboticDischarge.reasonCode in "Patient Refusal")
      and (NoAntithromboticDischarge.isCommunity() or NoAntithromboticDischarge.isDischarge())
      and NoAntithromboticDischarge.status in { 'active', 'completed' }
      and NoAntithromboticDischarge.intent = 'order'</code></pre>

  <p>For a more complete discussion of representation of negation within US Quality Core and quality improvement artifacts, see the <a href="general-requirements.html#negation-in-us-quality-core">Negation in US Quality Core</a> topic.</p>

  <a name="search-expectations"><b>Search Expectations</b></a>
  <p>For MedicationRequest, US Quality Core includes <code>patient</code>, <code>intent</code>, and <code>do-not-perform</code> search parameters, with required support for the <code>patient+intent</code> and <code>patient+intent+do-not-perform</code> combinations. The <code>do-not-perform</code> search supports retrieval of Medication Not Requested records. US Core search expectations not repeated here, such as searches involving <code>status</code>, remain part of applicable US Core conformance but are not called out as additional US Quality Core-specific search requirements.</p>
  <p>See the <a href="CapabilityStatement-us-quality-core-server.html">US Quality Core Server CapabilityStatement</a> for the computable conformance expectations and the <a href="relationship-with-uscore-qicore.html#search-expectations-and-us-core">Relationship with US Core and QI-Core</a> page for the search selection rules.</p>
