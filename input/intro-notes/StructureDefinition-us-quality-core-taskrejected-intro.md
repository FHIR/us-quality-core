  <a name="search-expectations"><b>Search Expectations</b></a>
  <p>For Task, US Quality Core includes <code>patient</code>, <code>status</code>, and <code>code</code> search parameters, with required support for the <code>patient+status</code> and <code>patient+code</code> combinations. The <code>status</code> search supports retrieval of Task Rejected records and other status-sensitive quality workflows. US Core 6.1.0 does not define a Task profile, so these search expectations are defined directly by US Quality Core.</p>
  <p>See the <a href="CapabilityStatement-us-quality-core-server.html">US Quality Core Server CapabilityStatement</a> for the computable conformance expectations and the <a href="relationship-with-uscore-qicore.html#search-expectations-and-us-core">Relationship with US Core and QI-Core</a> page for the search selection rules.</p>
