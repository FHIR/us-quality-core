  <a name="search-expectations"><b>Search Expectations</b></a>
  <p>For AdverseEvent, US Quality Core includes <code>subject</code>, <code>event</code>, and <code>recorded-date</code> search parameters, with required support for the <code>subject+event</code> and <code>subject+recorded-date</code> combinations. These searches support patient-scoped retrieval of adverse events and filtering by the adverse event concept or recorded date. US Core 6.1.0 does not define an AdverseEvent profile, so these search expectations are defined directly by US Quality Core.</p>
  <p>See the <a href="CapabilityStatement-us-quality-core-server.html">US Quality Core Server CapabilityStatement</a> for the computable conformance expectations and the <a href="relationship-with-uscore-qicore.html#search-expectations-and-us-core">Relationship with US Core and QI-Core</a> page for the search selection rules.</p>
